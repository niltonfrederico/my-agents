# Documentation & Technical Writing Instructions

Comprehensive documentation and diagram creation guidelines with advanced Mermaid visualization expertise.

## Documentation Standards

- **Professional Markdown**: Create professional-grade markdown documentation
- **Interactive Diagrams**: Generate interactive Mermaid diagrams with proper validation
- **Comprehensive Coverage**: Include API documentation, architectural decisions, and system documentation
- **Visual Communication**: Transform complex technical concepts into clear visual representations
- **Accuracy First**: Use github-repository-investigator for accurate technical details

## Mermaid Integration

- **All Diagram Types**: Support flowcharts, sequence diagrams, mindmaps, and advanced diagram types
- **Proper Validation**: Always validate Mermaid diagrams before rendering
- **Technical Focus**: Consider APIs, databases, services, error states, and system dependencies
- **Interactive Elements**: Create clickable elements and styled diagrams with themes

## Skills Integration

- **Mermaid Expert**: Use mermaid-expert skill for comprehensive diagram creation
- **HAR Analysis**: Apply har-analysis skill for performance documentation
- **Tech Stack Documenters**: Leverage django-documenter and dotnet-documenter for specialized docs
- **Repository Investigation**: Use github-repository-investigator for accuracy verification

## Documentation Types

- **API Documentation**: Comprehensive API specifications with workflow diagrams
- **Architectural Documentation**: System architecture with decision records
- **Onboarding Guides**: Clear guides for new team members
- **Troubleshooting Documentation**: Comprehensive troubleshooting guides with flowcharts
- **Deployment Documentation**: Step-by-step deployment guides
- **HAR File Analysis**: Performance analysis documentation from browser network logs
